현재 버전을 정식 v1.0으로 백업하고, 이후 개선 작업은 v1.1 작업으로 진행해줘.

1단계: 현재 버전 v1.0 백업

현재 상태는 사용 가능한 v1.0 버전이다.
먼저 현재 상태를 GitHub에 백업하고 태그를 만들어서 언제든 돌아갈 수 있게 해줘.

작업:
1. git status로 변경 파일을 확인해줘.
2. 현재 상태를 docs\작업로그.md에 “v1.0 사용 가능 버전 백업”으로 기록해줘.
3. 현재 테스트 결과를 docs\테스트결과.md에 기록해줘.
4. git add . 실행해줘.
5. 아래 커밋 메시지로 commit 해줘.

commit message:
release v1.0 working single-page pdf converter

6. v1.0 태그를 생성해줘.

tag:
v1.0

7. git push 해줘.
8. git push origin v1.0 으로 태그도 push 해줘.
9. push 완료 후 현재 브랜치명, 커밋 해시, 태그명을 알려줘.

주의:
- v1.0 백업 단계에서는 기능 수정하지마.
- 현재 상태 그대로 백업만 해줘.

2단계: v1.1 개선 작업 시작

v1.0 백업이 완료되면, 이후 작업은 v1.1 개선 작업으로 진행해줘.

현재 방식은 전체 페이지를 PNG로 캡처한 뒤 PDF 1페이지로 변환하는 구조다.
기능은 유지하되, 배경색 차이와 화질 문제를 개선해줘.

문제:
1. 캡처 이미지의 배경색과 PDF 페이지 배경색이 약간 달라서 경계가 보인다.
2. PDF가 이미지 기반이라 확대하면 화질이 좋지 않다.

수정 목표:
1. PDF 페이지 배경색과 캡처 PNG 배경색을 동일하게 맞춰라.
2. 가능하면 최종 PDF 배경을 white(#ffffff)로 채우고 이미지를 올려라.
3. PNG에 투명 영역이 있으면 흰색 배경으로 flatten 처리해라.
4. Playwright screenshot 품질을 높이기 위해 device_scale_factor를 2로 적용해라.
5. 필요하면 사용자가 품질 옵션을 선택할 수 있게 해라.
   - 보통: scale 1
   - 고화질: scale 2
   - 최고화질: scale 3
6. 최종 PDF 폭은 사용자가 선택한 용지 너비 기준으로 유지해라.
   예: A4 794px
7. scale이 2여도 PDF 표시 폭은 794px로 유지하고, 내부 이미지만 고해상도로 사용해라.
8. PDF는 계속 1페이지여야 한다.
9. 하단 crop과 좌우 여백 보정 기능은 유지해라.
10. page.pdf()는 사용하지 마라.

검증:
- scale 1, scale 2 결과 PDF를 각각 생성해 비교해라.
- PDF 페이지 수가 1인지 확인해라.
- 최종 이미지 크기와 PDF 페이지 크기를 출력해라.
- 파일 크기 증가량도 기록해라.
- docs\테스트결과.md에 기록해라.

주의:
- v1.1 개선 작업은 git commit/push 하지마.
- v1.1 수정 결과는 테스트 결과와 변경 파일 목록만 알려줘.
- v1.0으로 되돌리는 방법도 마지막에 알려줘.
