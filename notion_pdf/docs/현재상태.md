# 현재상태

작성일: 2026-05-16

## v3.1 실험 버전 백업 상태

현재 v3.1은 OCR/DOM 텍스트 레이어와 Notion 상단 배너 제거를 실험한 버전이지만, 사용자 검증 기준으로 PDF 생성 기능이 깨진 상태다.
일부 결과가 거의 빈 페이지처럼 생성될 수 있어 안정 버전으로 사용하지 않는다.
이 상태는 `v3.1` 태그로 백업한 뒤, main 브랜치는 v3.0 안정 버전으로 복구한다.

## 1. 프로젝트 개요

프로젝트 경로:

```text
~/git/dev/projects/Tilon/notion_pdf
```

목적:

Notion HTML 파일 또는 공개 URL을 Playwright Chromium으로 렌더링하고, `page.pdf()` 없이 전체 화면 PNG screenshot을 만든 뒤 `img2pdf`로 긴 1페이지 PDF를 생성한다.

현재 핵심 원칙:

- `page.pdf()`는 사용하지 않는다.
- 최종 PDF는 계속 1페이지여야 한다.
- 하단 crop을 유지한다.
- 좌우 여백 보정을 유지한다.
- 고화질 scale 2/3 캡처를 유지한다.
- 가능한 경우 OCR보다 DOM 원본 텍스트를 우선해 텍스트 레이어를 만든다.

## 3. 문서 인코딩 정책

문서 인코딩 정책: UTF-8

Mac과 Windows에서 문서가 동일하게 보이도록 Markdown과 TXT 파일을 UTF-8, LF 줄바꿈으로 통일한다.

`.gitattributes` 정책:

```text
*.md text working-tree-encoding=UTF-8 eol=lf
*.txt text working-tree-encoding=UTF-8 eol=lf
*.py text eol=lf
*.html text eol=lf
*.css text eol=lf
*.js text eol=lf
```

검증 방법:

```bash
python - <<'PY'
from pathlib import Path
for p in list(Path('docs').glob('*.md')) + [Path('README.md')]:
    text = p.read_text(encoding='utf-8')
    print('OK UTF-8:', p, len(text))
PY
```

## 5. 텍스트 레이어 정책

현재 v3.1 복구 기준은 v3.0의 긴 1페이지 PDF 생성 안정성을 우선으로 유지하면서, 기본 텍스트 레이어는 DOM 원문 우선 + OCR 보조 방식으로 적용한다.

텍스트 레이어 모드:

```text
hybrid : DOM 원문 + OCR 보조, 기본값
dom    : DOM 원문 텍스트 기반
ocr    : 이미지 기반 OCR 텍스트
none   : 텍스트 레이어 없음
```

기본 권장 모드:

```text
hybrid
```

정책:

- 기본값은 `hybrid`다.
- DOM 텍스트를 우선 삽입하고, DOM 영역과 겹치는 OCR 텍스트는 제외한다.
- OCR은 DOM으로 잡히지 않는 이미지 내부 텍스트 보조 용도다.
- 복사 결과가 깨지는 경우 DOM/OCR 중복 삽입 여부를 먼저 확인한다.
- 텍스트 레이어 개선보다 긴 1페이지 PDF 보기 품질, crop, 여백 보정, 고화질 캡처 안정성을 우선한다.

DOM 기반 처리:

1. Playwright로 Notion 페이지 또는 HTML 파일을 렌더링한다.
2. DOM text node를 순회한다.
3. 각 text node의 원본 `textContent`를 수집한다.
4. `Range.getClientRects()` 기준 줄 위치를 수집한다.
5. computed style에서 font size, font weight, line height를 수집한다.
6. screenshot scale, crop left, paste x, rebalance shift 값을 이용해 최종 PNG 좌표로 변환한다.
7. 최종 1페이지 PDF 좌표로 다시 변환한다.
8. pypdf로 invisible text layer를 추가한다.

Hybrid 처리:

- DOM 텍스트는 원본 문자열이므로 영어 대문자, Windows 경로, PowerShell 명령어, 한글+영문 혼합 문장에 우선 적용한다.
- OCR은 이미지 안의 텍스트, 캡처 이미지, 표 이미지처럼 DOM text node로 잡히지 않는 영역에 보조로 적용한다.
- OCR word의 중심점이 DOM 텍스트 박스와 겹치면 중복 삽입을 피한다.
- 복사 결과가 섞이면 DOM/OCR 중복 또는 OCR y-band 마스킹 실패로 판단한다.

Notion 상단 배너 제거:

- 공개 Notion URL 변환 시 캡처 전에 가입/로그인 유도 배너를 숨긴다.
- 제거 대상 문구:
  - `You're almost there`
  - `sign up to start building in Notion today`
  - `Sign up or login`
- `role=banner`, `header`, `nav`, fixed/sticky top 영역을 제거 후보로 본다.
- 문서 본문이나 실제 제목은 제거하지 않도록 top chrome 영역 중심으로 처리한다.
- 제거 로그는 `output/notion_banner_cleanup.json`에 저장한다.

## 6. OCR 구조

OCR 언어:

```text
kor+eng
```

OCR 방식:

```text
최종 고해상도 PNG
-> 높이 기준 chunk 분할
-> chunk별 전처리
-> Tesseract TSV OCR
-> 좌표를 최종 PDF 좌표로 변환
-> invisible text layer 추가
```

OCR 전처리 후보:

- grayscale
- contrast enhancement
- sharpen
- threshold/binarize

chunk 기준:

- 짧은 문서는 chunk 1개로 OCR한다.
- 긴 문서는 높이 기준 여러 chunk로 나눠 OCR한다.
- 기본 chunk 높이는 `OCR_CHUNK_HEIGHT_PX=3000`이다.
- 기본 overlap은 `OCR_CHUNK_OVERLAP_PX=80`이다.
- OCR 실패 구간은 `ocr_failed_chunks`에 chunk 번호와 위치를 기록한다.

현재 한계:

- DOM에 없는 이미지 내부 텍스트는 OCR에 의존한다.
- OCR 보조 영역은 OCR 엔진 특성상 일부 오인식될 수 있다.
- DOM 좌표 매핑은 crop/recenter를 반영하지만, PDF 뷰어의 선택 하이라이트 위치가 시각적으로 완벽히 일치하지 않을 수 있다.

## 10. Cloudflare Tunnel 공개 방식

현재 개발/테스트 공개 구조:

```text
Mac mini
-> Flask 서버 실행
-> Cloudflare Tunnel 연결
-> 외부 HTTPS 주소 생성
-> 다른 PC/사용자 접근 가능
```

Cloudflare Tunnel 실행:

```bash
cloudflared tunnel --url http://127.0.0.1:5000
```

성공 시 표시되는 주소 예시:

```text
https://xxxxx.trycloudflare.com
```

주의사항:

- Flask 서버 터미널이 종료되면 외부 접속이 불가능하다.
- `cloudflared` 터미널이 종료되면 외부 접속이 불가능하다.
- `trycloudflare.com` 주소는 임시 주소이며 재실행 시 바뀔 수 있다.
- Mac 절전 모드 진입 시 연결이 끊길 수 있다.
- OCR 및 고화질 PDF 생성은 CPU/RAM 사용량이 크다.
