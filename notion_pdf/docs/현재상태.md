# 현재 상태

작성일: 2026-05-17

## 안정 기준

- 현재 안정 기준은 `v3.0`이다.
- 긴 1페이지 PDF 생성, 하단 crop, 좌우 여백 보정, 고화질 캡처는 `v3.0` 구조를 유지해야 한다.
- `page.pdf()`는 사용하지 않는다.
- `full_page=True`에 의존하지 않는다.
- `v3.1` 실험 변경은 복사 품질 문제와 PDF 생성 안정성 저하가 있었으므로 안정 기능보다 우선하지 않는다.

## 현재 핵심 기능

- Notion 공개 URL 또는 HTML 파일을 입력받는다.
- Playwright Chromium으로 페이지를 렌더링한다.
- 캡처 전에 Notion 토글 자동 펼치기 옵션을 적용한다.
- 캡처 전에 Notion 상단 로그인/가입 배너 제거 옵션을 적용한다.
- screenshot `clip` 기반으로 긴 PNG를 생성한다.
- PNG 하단 빈 영역을 crop한다.
- 좌우 여백을 보정해 콘텐츠를 중앙 정렬한다.
- `img2pdf`로 긴 1페이지 PDF를 만든다.
- `pypdf`로 PDF가 1페이지인지 검증한다.
- 텍스트 레이어 옵션은 `none`, `ocr`, `dom`, `hybrid`를 지원한다.
- Notion 공개 페이지 상단 로그인/가입 배너는 캡처 전에 숨긴다.
- Cloudflare Tunnel로 외부 HTTPS 접근이 가능하다.

## Notion 전처리 기능

기본값:

- Notion 토글 자동 펼치기: 켜짐
- Notion 상단 배너 제거: 켜짐

토글 자동 펼치기:

- `details:not([open])`는 `open` 상태로 변경한다.
- `aria-expanded="false"`인 Notion toggle 후보를 반복적으로 클릭한다.
- 중첩 토글을 고려해 여러 pass를 수행한다.
- 토글 확장 후 DOM 높이를 다시 측정하고 최종 캡처 높이에 반영한다.
- 실패해도 PDF 생성은 중단하지 않고 로그만 남긴다.

상단 배너 제거:

- `You're almost there`
- `sign up to start building in Notion today`
- `Sign up`
- `login`

위 문구가 포함된 `header`, `nav`, `role=banner`, fixed/sticky top 영역을 숨긴다. 실제 문서 본문 컨테이너는 제거하지 않도록 보호한다.

한계:

- Notion DOM 구조가 바뀌면 일부 토글이 자동 탐지되지 않을 수 있다.
- 일반 버튼을 무작정 클릭하지 않도록 보수적으로 판단하므로, 커스텀 토글 형태는 남을 수 있다.
- 관리자 페이지와 다운로드 라우트는 아직 인증이 없다.

## 상시 운영 구조

이번 작업에서 상시 운영을 위한 기본 구조를 추가했다.

- Flask 서버는 launchd로 백그라운드 자동 실행할 수 있다.
- Cloudflare quick tunnel도 launchd로 자동 실행할 수 있다.
- Mac 재부팅 후 launchd가 Flask 서버와 tunnel을 다시 실행하도록 구성할 수 있다.
- quick tunnel은 임시 주소가 바뀔 수 있으므로 장기 운영에는 named tunnel과 고정 도메인 전환이 필요하다.

관련 파일:

- `scripts/macos/start_flask.sh`
- `scripts/macos/start_cloudflare_quick_tunnel.sh`
- `scripts/macos/com.notion_pdf.flask.plist`
- `scripts/macos/com.notion_pdf.cloudflare.quick.plist`
- `scripts/macos/com.notion_pdf.cleanup.plist`

## 데이터베이스 상태

SQLite 기반 변환 이력 저장 구조를 추가했다.

- DB 파일: `instance/notion_pdf.db`
- DB 모듈: `db.py`
- 관리자 페이지: `/admin/conversions`
- DB 저장 실패는 PDF 변환 실패로 전파하지 않고 로그만 남긴다.
- 현재 테이블명: `conversions`
- 요청 문구의 `conversion_history` 테이블은 현재 존재하지 않는다.
- 업로드 원본 파일 경로는 `input_file_path`에 기록한다.
- 업로드 원본 파일 크기는 `input_file_size`에 기록한다.
- 업로드 원본 파일명은 `input_original_name`에 기록한다.
- 결과 PDF 경로는 `output_pdf_path`에 기록한다.
- 결과 TXT 경로는 `output_txt_path`에 기록한다.
- 결과 PNG 경로는 `output_png_path`에 기록한다.

저장 항목:

- `id`
- `created_at`
- `source_type`
- `source_url`
- `original_filename`
- `output_pdf_path`
- `output_txt_path`
- `page_width`
- `margin`
- `quality_scale`
- `text_layer_mode`
- `status`
- `error_message`
- `file_size`
- `client_ip`
- `expires_at`

## 자동 삭제 정책

기본 보관 기간은 7일이다.

- `expires_at`이 지난 DB 기록을 삭제한다.
- 오래된 `output` PDF, PNG, TXT 파일을 삭제한다.
- 업로드 임시 폴더의 오래된 PDF, PNG, TXT 파일도 삭제한다.
- 삭제 실패는 앱을 중단하지 않고 `logs/cleanup.log`에 기록한다.

확인된 한계:

- 일반 변환 PDF/TXT는 현재 `output/` 폴더가 아니라 OS 임시 폴더의 `notion_pdf` 디렉터리에 저장된다.
- DB의 `output_pdf_path`는 실제 파일을 가리키며 파일 크기도 0보다 크다.
- `/output/<파일명>.pdf` 직접 접근 라우트는 없으며 HTTP 404가 반환된다.
- `/download/<job_id>`는 메모리 기반 job 정보에 의존하므로 서버 재시작 후 이전 job 다운로드는 불가능하다.
- `/admin/conversions`는 인증 없이 접근 가능하므로 외부 공개 시 보안 위험이 있다.
- 관리자 페이지에 서버 절대 파일 경로가 표시된다.

현재 판단:

- DB 저장, 관리자 목록 표시, cleanup 로직은 동작한다.
- output 폴더 직접 접근과 영구 다운로드 URL은 아직 구현되어 있지 않다.
- 외부 Cloudflare 주소는 로컬 Flask 라우트를 그대로 노출하므로, 관리자 페이지 인증이 다음 우선순위다.

## 파일 저장 정책

현재 저장 정책:

- 업로드 원본 파일: `uploads/YYYY-MM/`
- 변환 결과 PDF: `output/pdf/`
- 변환 결과 PNG: `output/png/`
- 추출 텍스트 TXT: `output/txt/`
- 디버그 이미지: `output/debug/`
- 테스트/검증 산출물: `output/tests/`

파일명 정책:

- `secure_filename`을 적용한다.
- timestamp, 원본 이름, 8자리 job id를 조합한다.
- 원본 파일명은 DB의 `input_original_name`에 따로 보관한다.

예:

```text
uploads/2026-05/20260519_133833_server-flow_2f0663d1.html
output/pdf/20260519_133833_server-flow_seamless_2f0663d1.pdf
output/png/20260519_133833_server-flow_seamless_2f0663d1.png
output/txt/20260519_133833_server-flow_seamless_2f0663d1.txt
```

허용 업로드 확장자:

- `html`
- `htm`
- `zip`
- `pdf`
- `txt`

현재 변환 지원:

- HTML/HTM 파일은 변환한다.
- PDF/ZIP/TXT는 원본 저장과 DB 실패 기록은 가능하지만, 긴 1페이지 PDF 변환은 아직 지원하지 않는다.

관리자 다운로드:

- 원본 파일: `/admin/conversions/<id>/download/input`
- 결과 PDF: `/admin/conversions/<id>/download/pdf`
- 결과 TXT: `/admin/conversions/<id>/download/txt`
- 결과 PNG: `/admin/conversions/<id>/download/png`

보안 주의:

- 관리자 페이지와 다운로드 라우트는 아직 인증이 없다.
- 다운로드 라우트는 DB에 기록된 파일 경로만 사용하고, `uploads/` 또는 `output/` 아래 파일만 허용한다.
- 외부 공개 시 `/admin/conversions` 접근 제한 또는 인증 추가가 필요하다.

cleanup 동작:

- 만료된 DB 기록의 `input_file_path`, `output_pdf_path`, `output_txt_path`, `output_png_path` 파일을 삭제한다.
- `uploads/`와 `output/` 아래 오래된 관리 대상 파일을 재귀적으로 삭제한다.
- 삭제 결과는 `logs/cleanup.log`에 기록한다.

## 긴 페이지 안정화 구조

v3.6 기준 긴 페이지 안정화 구조:

- 단일 대형 screenshot 대신 내부 chunk 캡처를 사용한다.
- 기본 chunk 높이는 `SCREENSHOT_CHUNK_HEIGHT_PX=8000`이다.
- 각 chunk를 viewport 캡처로 생성한 뒤 Pillow로 하나의 긴 PNG로 stitch한다.
- 최종 PDF는 계속 1페이지다.
- chunk 실패 시 `SCREENSHOT_CHUNK_RETRIES` 횟수만큼 재시도한다.
- 실패하면 chunk index, y 위치, 높이, 에러를 로그로 출력한다.

캡처 전 안정화:

- `document.fonts.ready` 대기
- 이미지 `complete` 및 `naturalWidth > 0` 확인
- lazy-loaded 이미지 로딩을 위한 페이지 강제 스크롤
- DOM 높이 안정화 대기
- table/database 주변 overflow clipping 완화 스타일 적용

debug:

- chunk PNG: `output/debug/chunks_*`
- clipping 후보 로그: `output/debug/*_clipping_blocks.json`
- 최종 debug PNG: `output/debug/debug_fullpage.png`

한계:

- Notion 가상 스크롤 database는 DOM에 실제 렌더링되지 않은 행까지 복원할 수 없다.
- 외부 iframe, 권한이 필요한 이미지, 브라우저가 로드하지 못한 원격 이미지는 캡처되지 않을 수 있다.
- chunk stitch는 안정성을 높이지만 매우 큰 문서에서는 메모리 사용량이 증가한다.

수동 실행:

```bash
python cleanup_old_records.py
```

dry-run:

```bash
python cleanup_old_records.py --dry-run
```

## 문서 인코딩 정책

- 모든 Markdown 문서는 UTF-8로 저장한다.
- Windows에서도 한글이 깨지지 않아야 한다.
- `.gitattributes` 정책:

```text
*.md text working-tree-encoding=UTF-8 eol=lf
*.txt text working-tree-encoding=UTF-8 eol=lf
*.py text eol=lf
*.html text eol=lf
*.css text eol=lf
*.js text eol=lf
```

검증 명령:

```bash
python - <<'PY'
from pathlib import Path
for p in list(Path('docs').glob('*.md')) + [Path('README.md')]:
    text = p.read_text(encoding='utf-8')
    print('OK UTF-8:', p, len(text))
PY
```

## 금지사항

- `page.pdf()` 사용 금지
- `full_page=True`에만 의존 금지
- 긴 1페이지 PDF 생성 기능 깨뜨리기 금지
- OCR/DOM 개선 때문에 PDF가 빈 페이지가 되면 즉시 중단
- 기능 깨짐 상태를 성공 처리하지 말 것
- 외부 공개 시 민감정보가 포함된 문서를 업로드하지 말 것
