## 2026-05-16 - OCR comparison after install

- Mac 환경에서 `ocrmypdf`와 `tesseract` 설치 상태를 확인했다.
  - `ocrmypdf`: `/opt/homebrew/bin/ocrmypdf`
  - `tesseract`: `/opt/homebrew/bin/tesseract`
  - `kor`, `eng` 언어 데이터 확인 완료
- OCR 적용 PDF와 비적용 PDF를 같은 HTML 샘플에서 각각 생성하는 비교 스크립트를 추가했다.
  - `tests/run_ocr_comparison.py`
  - 비적용 PDF: `output/ocr_comparison_no_ocr.pdf`
  - 적용 PDF: `output/ocr_comparison_with_ocr.pdf`
- 검증 대상은 Windows 경로, PowerShell 명령어, 코드블록이다.
- OCR 비적용 PDF는 pypdf 텍스트 추출 결과 0자로, 텍스트 선택/복사가 불가능한 이미지 PDF 상태를 확인했다.
- OCR 적용 PDF는 `ocr_status=applied`이며 텍스트 추출 결과가 생성되어 선택/복사 가능한 텍스트 레이어가 생겼다.
- 기본 `kor+eng` OCR에서 Windows 경로와 일부 명령/코드 라인은 정확히 복사 가능했다.
- `$env:PORT=5055`의 `0`이 `@`로 인식되거나, `python -m ... -s ...`의 `s`가 `5`로 인식되는 등 일부 코드/명령 특수문자 정확도 한계가 확인됐다.
- `OCR_LANG=eng`, `--oversample 300`, `--tesseract-pagesegmode 6` 실험도 수행했으나 항목별로 정확도 개선과 부작용이 섞여 기본 옵션을 유지했다.
