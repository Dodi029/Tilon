## 주요 파일 설명

### `app.py`

전체 애플리케이션의 핵심 파일이다.

포함 기능:

- Flask 서버
- HTML UI
- `/convert/url`
- `/convert/file`
- `/job/<job_id>`
- `/download/<job_id>`
- Playwright 렌더링
- PNG screenshot 캡처
- crop/padding/flatten
- `img2pdf` 변환
- `pypdf` 검증

### `requirements.txt`

Python 의존성:

- Flask
- Playwright
- beautifulsoup4
- requests
- pypdf
- Pillow
- img2pdf

### `tests/test_single_page_pdf.py`

단위 테스트.

검증 내용:

- 긴 HTML이 1페이지 PDF로 변환되는지
- PDF page count가 1인지
- crop과 좌우 여백 보정이 유지되는지
- scale 1과 scale 2 결과 비교
- scale 2 final PNG가 고해상도인지
- scale 2에서도 PDF page width가 794px인지

### `tests/run_server_pdf_flow.py`

통합 테스트.

검증 흐름:

1. 테스트 스크립트가 Flask 서버를 실행한다.
2. 서버 URL이 응답하는지 확인한다.
3. 샘플 HTML을 `/convert/file`로 업로드한다.
4. `/job/<job_id>`를 polling한다.
5. PDF를 다운로드한다.
6. `pypdf`로 1페이지 검증한다.
7. 최종 PNG/PDF 크기와 scale 정보를 출력한다.

## 현재 PDF 생성 핵심 구조

```text
HTML/URL
-> Playwright Chromium render
-> page.screenshot(..., clip=...)
-> white background flatten
-> pixel content bounding box
-> bottom crop
-> left/right margin correction
-> img2pdf
-> pypdf page count check
```

## 실행 명령 모음

서버:

```bash
python app.py
```

단위 테스트:

```bash
python -m unittest discover -s tests
```

통합 테스트:

```bash
python tests\run_server_pdf_flow.py
```

`page.pdf()` 미사용 확인:

```bash
rg "page\.pdf|full_page=True" app.py tests
```
