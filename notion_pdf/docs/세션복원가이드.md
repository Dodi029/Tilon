안정 기준:

- `v3.0`은 안정 버전이다.
- `v3.1`은 실험 중 또는 복구 대상일 수 있다.
- 안정 기능은 `v3.0` 기준으로 유지해야 한다.

핵심 기능:

- Notion URL 또는 HTML 파일 입력
- Playwright 렌더링
- `page.pdf()` 미사용
- `full_page=True` 의존 금지
- screenshot `clip` 기반 캡처
- PNG 기반 긴 1페이지 PDF 생성
- 하단 crop
- 좌우 여백 보정
- 고화질 캡처 옵션
- 텍스트 레이어 옵션
- Cloudflare Tunnel을 통한 외부 접근

금지사항:

- `page.pdf()` 사용 금지
- `full_page=True`에만 의존 금지
- 기존 긴 1페이지 PDF 기능 깨뜨리기 금지
- `v3.0` 안정 기능보다 OCR 개선을 우선하지 말 것
- OCR/DOM 수정 시 PDF가 빈 페이지가 되면 즉시 중단
- 기능 깨짐 상태를 성공 처리하지 말 것

현재 해결 중인 문제:

- Notion 상단 로그인/가입 배너 제거
- 일부 텍스트가 선택되지 않는 문제
- DOM 텍스트 레이어 누락 문제
- OCR/DOM 중복 삽입 시 복사 결과가 깨지는 문제
- Windows에서 Markdown 문서 한글 인코딩 깨짐 문제

처음 읽어야 할 문서:

- `docs/현재상태.md`
- `docs/작업로그.md`
- `docs/테스트결과.md`
- `docs/세션복원가이드.md`
- `docs/AI작업인수인계.md`
- `docs/Gemini_CLI_인수인계.md`
- `README.md`

테스트 명령:

```bash
python -m py_compile app.py
python -m unittest discover -s tests
python tests/run_server_pdf_flow.py
python tests/run_ocr_comparison.py
python tests/run_banner_text_validation.py
rg "page\.pdf|full_page=True" app.py tests
```

문서 인코딩 정책:

- 모든 Markdown 문서는 UTF-8이다.
- Windows에서도 깨지지 않아야 한다.
- `.gitattributes` 정책을 확인한다.
- `docs/*.md` UTF-8 읽기 테스트를 수행한다.

작업 로그 정책:

- 앞으로 어떤 AI CLI가 작업하든 `docs/작업로그.md`에 기록한다.
- 테스트 결과는 `docs/테스트결과.md`에 기록한다.
- 현재 상태 변경은 `docs/현재상태.md`에 기록한다.
- 새 세션 인수인계 내용은 `docs/AI작업인수인계.md`에 기록한다.

## 복원 절차

### 1. 저장소 받기

Windows:

```powershell
mkdir C:\dev
mkdir C:\dev\projects
cd C:\dev\projects
git clone https://github.com/Dodi029/Tilon.git
cd C:\dev\projects\Tilon\notion_pdf
```

Mac/Linux:

```bash
mkdir -p ~/dev/projects
cd ~/dev/projects
git clone https://github.com/Dodi029/Tilon.git
cd Tilon/notion_pdf
```

### 2. v2.0 상태 확인

```bash
git checkout main
git pull
git tag --list
git status --short
```

v2.0 태그로 정확히 이동하려면:

```bash
git checkout v2.0
```

### 3. Python 의존성 설치

```bash
python -m pip install -r requirements.txt
python -m playwright install chromium
```

### 4. 테스트

```bash
python -m unittest discover -s tests
python tests\run_server_pdf_flow.py
```

Mac/Linux에서는 두 번째 명령을 다음처럼 실행한다.

```bash
python tests/run_server_pdf_flow.py
```

### 5. 서버 실행

```bash
python app.py
```

접속:

```text
http://localhost:5000
```

### 6. Cloudflare Tunnel로 외부 공개

Mac에서 Cloudflare Tunnel을 설치한다.

```bash
brew install cloudflared
```

설치 확인:

```bash
cloudflared --version
```

Flask 서버가 실행 중인 상태에서 새 터미널을 열고 다음 명령을 실행한다.

```bash
cloudflared tunnel --url http://127.0.0.1:5000
```

성공하면 다음 형식의 임시 HTTPS 주소가 표시된다.

```text
https://xxxxx.trycloudflare.com
```

이 주소를 다른 PC, Windows 노트북, 외부 사용자에게 전달하면 현재 Mac 개발 서버에 접근할 수 있다.

주의:

- Flask 서버 터미널이 종료되면 외부 접속이 불가능하다.
- `cloudflared` 터미널이 종료되면 외부 접속이 불가능하다.
- `trycloudflare.com` 주소는 임시 주소이며 재실행 시 바뀔 수 있다.
- Mac 절전 모드 진입 시 연결이 끊길 수 있다.
- 현재 구조는 개발/테스트 목적이다.
- OCR 및 고화질 PDF 생성은 CPU/RAM 사용량이 크다.

Mac 절전 방지 권장 설정:

- 자동 잠자기 비활성화
- 디스플레이만 끄고 시스템은 유지
