## 새 Codex 세션에서 가장 먼저 읽을 문서

다음 순서로 읽는다.

1. `docs/현재상태.md`
2. `docs/프로젝트구조.md`
3. `docs/개발환경구조.md`
4. `docs/테스트결과.md`
5. `docs/작업로그.md`
6. `README.md`

## 복원 절차

### 1. 저장소 받기

Windows:

```powershell
mkdir C:\dev
mkdir C:\dev\projects
cd C:\dev\projects
git clone https://github.com/Dodi029/Tilon.git
cd C:\dev\projects\Tilon\notion_pdf
```

Mac/Linux:

```bash
mkdir -p ~/dev/projects
cd ~/dev/projects
git clone https://github.com/Dodi029/Tilon.git
cd Tilon/notion_pdf
```

### 2. v2.0 상태 확인

```bash
git checkout main
git pull
git tag --list
git status --short
```

v2.0 태그로 정확히 이동하려면:

```bash
git checkout v2.0
```

### 3. Python 의존성 설치

```bash
python -m pip install -r requirements.txt
python -m playwright install chromium
```

### 4. 테스트

```bash
python -m unittest discover -s tests
python tests\run_server_pdf_flow.py
```

Mac/Linux에서는 두 번째 명령을 다음처럼 실행한다.

```bash
python tests/run_server_pdf_flow.py
```

### 5. 서버 실행

```bash
python app.py
```

접속:

```text
http://localhost:5000
```

## 새 Codex 세션에 붙여넣을 복원 메시지

아래 메시지를 새 세션에 그대로 붙여넣으면 된다.

```text
현재 프로젝트는 C:\dev\projects\Tilon\notion_pdf 입니다.

먼저 아래 문서를 순서대로 읽고 현재 상태를 파악하세요.
1. docs\현재상태.md
2. docs\프로젝트구조.md
3. docs\개발환경구조.md
4. docs\테스트결과.md
5. docs\작업로그.md
6. README.md

현재 프로젝트 목적:
Notion HTML 또는 공개 URL을 Playwright로 렌더링하고, page.pdf() 없이 PNG screenshot 기반으로 긴 1페이지 PDF를 생성하는 Flask 도구입니다.

현재 구현 방식:
- Flask 서버: app.py
- 실행: python app.py
- 접속: http://localhost:5000
- PDF 생성: Playwright screenshot clip -> PNG white flatten -> bottom crop -> 좌우 여백 보정 -> img2pdf -> pypdf 검증
- page.pdf()는 사용하지 않습니다.
- 기본 PDF page width는 794px입니다.
- 기본 scale은 2이며, 고해상도 PNG를 사용하되 PDF 표시 width는 794px로 유지합니다.

현재 Git 상태:
- 브랜치: main
- v1.0 태그는 안정 백업 버전입니다.
- v2.0 태그는 안정 long single-page PDF converter 버전입니다.
- 원격 저장소: https://github.com/Dodi029/Tilon.git

테스트 명령:
- python -m unittest discover -s tests
- python tests\run_server_pdf_flow.py

작업 규칙:
- page.pdf()를 다시 도입하지 마세요.
- 하단 crop과 좌우 여백 보정 구조를 유지하세요.
- PDF는 계속 1페이지여야 합니다.
- 기능 수정 전 git status를 확인하세요.
- 중요한 변경 후 docs\작업로그.md와 docs\테스트결과.md를 갱신하세요.
```

## 복구 명령

v1.0으로 이동:

```bash
git checkout v1.0
```

v2.0으로 이동:

```bash
git checkout v2.0
```

main 최신으로 복귀:

```bash
git checkout main
git pull
```

작업 트리를 v2.0으로 강제 복구:

```bash
git reset --hard v2.0
```

주의: `git reset --hard`는 커밋되지 않은 변경사항을 삭제한다.
