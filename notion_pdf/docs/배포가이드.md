# 배포 가이드

작성일: 2026-05-16

이 문서는 현재 Mac 개발 환경에서 Cloudflare Tunnel을 이용해 로컬 Flask 서버를 외부 HTTPS 주소로 공개하는 방법을 정리한다. 현재 구조는 개발/테스트 목적이며, 운영 배포 구조가 아니다.

## 현재 공개 구조

```text
Mac mini
→ Flask 서버 실행
→ Cloudflare Tunnel 연결
→ 외부 HTTPS 주소 생성
→ 다른 PC/사용자 접근 가능
```

Mac mini가 메인 개발 서버 역할을 하고, Flask 앱이 `127.0.0.1:5000`에서 실행된다. Cloudflare Tunnel은 이 로컬 서버를 임시 HTTPS 주소인 `https://xxxxx.trycloudflare.com` 형태로 외부에 공개한다.

## 현재 프로젝트 기준 서버 실행 방법

프로젝트 경로:

```bash
cd ~/git/dev/projects/Tilon/notion_pdf
```

가상환경 활성화:

```bash
source .venv/bin/activate
```

서버 실행:

```bash
python app.py
```

기본 접속 주소:

```text
http://127.0.0.1:5000
```

## Cloudflare Tunnel 설치 방법

Mac:

```bash
brew install cloudflared
```

설치 확인:

```bash
cloudflared --version
```

## Cloudflare Tunnel 실행 방법

Flask 서버가 이미 실행 중인 상태에서 새 터미널을 연다.

새 터미널에서:

```bash
cloudflared tunnel --url http://127.0.0.1:5000
```

성공하면 터미널에 외부 접속용 HTTPS 주소가 표시된다.

성공 시 표시되는 주소 예시:

```text
https://xxxxx.trycloudflare.com
```

## 외부 접속 방법

다른 PC 또는 사용자에게 아래 형식의 주소를 전달한다.

```text
https://xxxxx.trycloudflare.com
```

외부 사용자는 브라우저에서 이 주소로 접속하면 현재 Mac mini에서 실행 중인 Flask 서버에 접근할 수 있다.

## 중요한 주의사항

1. Flask 서버 터미널이 종료되면 외부 접속이 불가능하다.
2. `cloudflared` 터미널이 종료되면 외부 접속이 불가능하다.
3. `trycloudflare.com` 주소는 임시 주소다.
4. `cloudflared`를 재실행하면 주소가 바뀔 수 있다.
5. Mac이 절전 모드에 들어가면 연결이 끊길 수 있다.
6. 현재 구조는 개발/테스트 목적이다.
7. OCR 및 고화질 PDF 생성은 CPU/RAM 사용량이 크다.

## Mac 절전 방지 권장 설정

시스템 설정:

- 자동 잠자기 비활성화
- 디스플레이만 끄고 시스템은 유지

장시간 외부 테스트를 진행할 때는 Mac mini가 절전 상태로 진입하지 않도록 전원 설정을 먼저 확인한다.
