# 배포 가이드

작성일: 2026-05-17

이 문서는 Mac mini 개발 서버에서 `notion_pdf`를 외부에서 계속 접근 가능한 형태로 운영하는 방법을 정리한다. 현재 구현은 v3.0 안정 PDF 생성 구조를 유지하면서 launchd, Cloudflare Tunnel, SQLite 이력 저장, 자동 삭제를 붙인 개발/소규모 운영 구조다.

## 현재 공개 구조

```text
Mac mini
→ Flask 서버 실행
→ Cloudflare Tunnel 연결
→ 외부 HTTPS 주소 생성
→ 다른 PC/사용자 접근 가능
```

## 프로젝트 기준 서버 실행

프로젝트 경로:

```bash
cd ~/git/dev/projects/Tilon/notion_pdf
```

가상환경 활성화:

```bash
source .venv/bin/activate
```

서버 실행:

```bash
python app.py
```

기본 접속 주소:

```text
http://127.0.0.1:5000
```

## Cloudflare Tunnel 설치

```bash
brew install cloudflared
cloudflared --version
```

## 개발용 quick tunnel

Flask 서버가 실행 중인 상태에서 새 터미널에서 실행한다.

```bash
cloudflared tunnel --url http://127.0.0.1:5000
```

성공 시 다음 형식의 주소가 표시된다.

```text
https://xxxxx.trycloudflare.com
```

이 주소를 다른 PC, Windows 노트북, 외부 사용자에게 전달하면 현재 Mac mini Flask 서버에 접속할 수 있다.

quick tunnel 특징:

- 설정이 가장 단순하다.
- Cloudflare 계정 없이도 빠르게 테스트할 수 있다.
- `cloudflared`를 재실행하면 주소가 바뀔 수 있다.
- 장기 운영이나 고정 주소가 필요한 경우에는 적합하지 않다.

## 운영용 named tunnel

named tunnel은 Cloudflare 계정, 도메인, tunnel 인증 정보를 사용하는 운영 방식이다.

장점:

- 고정 도메인을 사용할 수 있다.
- Cloudflare Zero Trust 정책과 연결할 수 있다.
- 서비스 재시작 후에도 같은 주소를 유지할 수 있다.
- 운영 로그와 접근 정책을 관리하기 쉽다.

향후 확장 예:

```bash
cloudflared tunnel login
cloudflared tunnel create notion-pdf
cloudflared tunnel route dns notion-pdf pdf.example.com
cloudflared tunnel run notion-pdf
```

현재는 quick tunnel을 유지하되, 장기 운영 단계에서 named tunnel로 전환하는 것을 권장한다.

## launchd 자동 실행

Mac 재부팅 후에도 Flask 서버와 Cloudflare Tunnel이 자동으로 다시 실행되도록 launchd 예시 파일을 추가했다.

관련 파일:

- `scripts/macos/start_flask.sh`
- `scripts/macos/start_cloudflare_quick_tunnel.sh`
- `scripts/macos/com.notion_pdf.flask.plist`
- `scripts/macos/com.notion_pdf.cloudflare.quick.plist`
- `scripts/macos/com.notion_pdf.cleanup.plist`

스크립트 실행 권한:

```bash
chmod +x scripts/macos/start_flask.sh scripts/macos/start_cloudflare_quick_tunnel.sh
```

Flask 자동 실행 등록:

```bash
cp scripts/macos/com.notion_pdf.flask.plist ~/Library/LaunchAgents/
launchctl load ~/Library/LaunchAgents/com.notion_pdf.flask.plist
```

Cloudflare quick tunnel 자동 실행 등록:

```bash
cp scripts/macos/com.notion_pdf.cloudflare.quick.plist ~/Library/LaunchAgents/
launchctl load ~/Library/LaunchAgents/com.notion_pdf.cloudflare.quick.plist
```

중지:

```bash
launchctl unload ~/Library/LaunchAgents/com.notion_pdf.flask.plist
launchctl unload ~/Library/LaunchAgents/com.notion_pdf.cloudflare.quick.plist
```

재시작:

```bash
launchctl unload ~/Library/LaunchAgents/com.notion_pdf.flask.plist
launchctl load ~/Library/LaunchAgents/com.notion_pdf.flask.plist
```

로그 확인:

```bash
tail -f /tmp/notion_pdf_flask.launchd.log
tail -f /tmp/notion_pdf_flask.launchd.err.log
tail -f /tmp/notion_pdf_cloudflared.quick.launchd.log
tail -f /tmp/notion_pdf_cloudflared.quick.launchd.err.log
```

주의: launchd plist 안의 프로젝트 경로는 `~/git/dev/projects/Tilon/notion_pdf` 기준이다. 실제 Mac 경로가 다르면 plist 안의 경로를 먼저 수정한다.

## 데이터베이스

SQLite DB를 사용한다.

```text
instance/notion_pdf.db
```

저장 정보:

- `id`
- `created_at`
- `source_type`: `url` 또는 `html_upload`
- `source_url`
- `original_filename`
- `output_pdf_path`
- `output_txt_path`
- `page_width`
- `margin`
- `quality_scale`
- `text_layer_mode`
- `status`: `success` 또는 `failed`
- `error_message`
- `file_size`
- `client_ip`
- `expires_at`

관리자 확인 페이지:

```text
http://127.0.0.1:5000/admin/conversions
```

DB 기능은 `db.py`로 분리했다. 나중에 MariaDB로 확장할 때는 `record_conversion`, `list_recent_conversions`, `delete_expired_records` 같은 함수의 구현체를 교체하는 방향으로 진행한다.

DB 기록 실패는 PDF 변환 자체를 중단하지 않는다.

## 자동 삭제 정책

기본 보관 기간은 7일이다.

- `expires_at`이 지난 DB 기록 삭제
- 만료 기록에 연결된 PDF/TXT 파일 삭제
- `output` 폴더의 오래된 PDF, PNG, TXT 파일 삭제
- 업로드 임시 폴더의 오래된 PDF, PNG, TXT 파일 삭제
- 삭제 실패 시 앱을 중단하지 않고 로그만 기록

수동 실행:

```bash
python cleanup_old_records.py
```

dry-run:

```bash
python cleanup_old_records.py --dry-run
```

보관 기간 변경:

```bash
python cleanup_old_records.py --retention-days 14
```

로그:

```text
logs/cleanup.log
```

launchd 하루 1회 실행 등록:

```bash
cp scripts/macos/com.notion_pdf.cleanup.plist ~/Library/LaunchAgents/
launchctl load ~/Library/LaunchAgents/com.notion_pdf.cleanup.plist
```

## 보안 및 운영 주의사항

1. 외부 공개 주소는 접근 가능한 모든 사용자가 변환 기능을 사용할 수 있다.
2. 관리자 페이지 `/admin/conversions`에는 별도 인증이 없으므로 공개 범위를 제한해야 한다.
3. 개인정보, 내부 문서, 민감정보가 포함된 파일은 업로드하지 않도록 안내해야 한다.
4. 업로드 크기 제한은 기본 `MAX_UPLOAD_MB=50`이다.
5. OCR과 고화질 캡처는 CPU/RAM 사용량이 크다.
6. output 파일 자동 삭제는 디스크 사용량과 민감정보 잔존 위험을 줄이기 위해 필요하다.
7. quick tunnel 주소는 임시 주소이며 재실행 시 바뀔 수 있다.
8. 운영 주소가 필요하면 named tunnel과 고정 도메인을 사용한다.
9. Mac mini가 절전 모드에 들어가면 Flask 서버와 tunnel 연결이 끊길 수 있다.

- 외부 접속 테스트
- PDF 결과 확인
