주의: 현재 변환 결과 PDF/TXT는 `output/` 폴더가 아니라 OS 임시 폴더의 `notion_pdf` 디렉터리에 저장된다.

```text

```

`output/` 폴더에는 테스트 산출물 PDF/PNG/TXT가 존재하지만, 일반 변환 성공 기록의 `output_pdf_path`는 현재 `output/` 폴더를 가리키지 않는다.

외부 직접 접근 확인:

```text
GET /output/<최근 PDF 파일명>.pdf -> 404
GET /download/<이전 job_id> -> 404
```

현재 구조에서는 `/output/<파일명>.pdf` 직접 접근 라우트가 없다. `/download/<job_id>`는 Flask 프로세스 메모리의 job 정보에 의존하므로 서버 재시작 후 이전 파일은 URL로 받을 수 없다.

Cloudflare Tunnel은 로컬 Flask 라우트를 그대로 외부에 노출하므로, 현재 상태에서는 `/output/<파일명>.pdf`도 외부에서 직접 접근되지 않는 구조로 판단한다. 반대로 `/admin/conversions`는 인증 없이 노출될 수 있으므로 보안 위험이 있다.

- DOM 기반 텍스트 레이어가 OCR보다 복사 정확도가 높다.
- OCR chunk 로그와 실패 chunk 정보가 결과에 남는다.
- v3.0 백업 기준으로 사용 가능하다.

## 2026-05-17 - v3.0 기준 최소 개선 검증

### 변경 내용

- 공개 Notion URL 캡처 전에 상단 로그인/가입 배너를 숨기도록 했다.
- 제거 대상 문구:
  - `You're almost there`
  - `sign up to start building in Notion today`
  - `Sign up or login`
- 제거 후보는 `role=banner`, `header`, `nav`, fixed/sticky top 요소로 제한했다.
- 실제 본문 컨테이너인 `main`, `article`, `.notion-page-content`, `.notion-page-content-inner`는 제거하지 않는다.
- DOM 텍스트 수집은 `TreeWalker`와 `Range.getClientRects()` 기반을 유지한다.
- Hybrid 모드에서는 DOM 텍스트와 같은 y-band에 있는 OCR 텍스트를 제외해 중복 삽입을 줄였다.
- `.gitattributes`를 추가해 Markdown/TXT UTF-8 정책을 기록했다.
- Gemini CLI 인수인계 문서를 추가했다.
