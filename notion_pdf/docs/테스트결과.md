## 2026-05-16 - dynamic chunk OCR improvement test

### 구현 확인

- 기존 `ocrmypdf` 단일 페이지 후처리 대신 최종 PNG를 높이 기준 chunk로 분할해 OCR한다.
- 각 chunk는 Tesseract TSV 결과를 사용한다.
- OCR 전용 이미지는 최종 고해상도 PNG를 기준으로 사용한다.
- chunk별 전처리 후보:
  - grayscale
  - contrast enhancement
  - sharpen
  - threshold/binarize
- 각 chunk에서 추출 텍스트 길이가 가장 좋은 전처리 결과를 선택한다.
- OCR 결과 좌표를 최종 1페이지 PDF 좌표로 변환해 invisible text layer를 추가한다.
- OCR 실패 또는 부족 상태여도 PDF 생성 자체는 실패시키지 않는다.
- `page.pdf()`는 사용하지 않는다.

## 2026-05-16 - DOM text layer comparison test

### 구현 확인

- PNG 캡처 기반 1페이지 PDF 생성 구조는 유지했다.
- `page.pdf()`는 사용하지 않는다.
- 텍스트 레이어 모드를 추가했다.
  - `none`: 텍스트 레이어 없음
  - `ocr`: OCR 기반 텍스트 레이어
  - `dom`: DOM 원본 텍스트 기반 텍스트 레이어
  - `hybrid`: DOM 기반 + DOM과 겹치지 않는 OCR 보조
- Playwright 렌더링 후 DOM text node에서 다음 정보를 수집한다.
  - `textContent`
  - `getBoundingClientRect()`
  - font size
  - font weight
  - line height
  - x, y, width, height
- crop/recenter 이후의 이미지 좌표 변환값을 이용해 DOM 좌표를 최종 1페이지 PDF 좌표로 매핑한다.
- OCR은 이미지 안의 텍스트나 DOM으로 잡히지 않는 영역을 위한 보조 수단으로 유지한다.
