\# Codex 기본 지시문



이 저장소는 사용자의 개인 개발 작업공간이다.



사용 목적:

\- 간단한 웹 프로그램 개발

\- Windows PowerShell 스크립트 개발

\- Windows Server 테스트 자동화

\- 작업 내용 문서화

\- GitHub commit/push 기반 백업

\- Mac과 Windows 노트북에서 동일한 개발환경 유지

\- 서버 배포 가능 구조 유지



작업 규칙:

1\. 작업 전 현재 프로젝트 구조를 먼저 확인한다.

2\. 기존 파일을 함부로 삭제하지 않는다.

3\. 비밀번호, 토큰, 인증서, 서버 접속정보를 코드에 직접 저장하지 않는다.

4\. 필요한 환경변수는 .env.example에만 예시로 작성한다.

5\. 기능 추가 후 실행 방법을 README.md 또는 docs에 기록한다.

6\. 오류가 발생하면 원인, 해결방법, 재현방법을 docs/테스트결과.md에 기록한다.

7\. 작업 완료 후 변경 파일 목록과 git diff 요약을 제공한다.

8\. 사용자가 승인하기 전에는 git push를 하지 않는다.

9\. 모든 작업 내용은 docs/작업로그.md에 날짜별로 기록한다.

10\. Windows 관련 스크립트는 scripts/powershell 또는 scripts/windows 아래에 작성한다.

11\. 웹 프로그램은 apps 아래에 작성한다.

12\. 서버 배포 관련 파일은 deploy 아래에 작성한다.



문서화 위치:

\- 요구사항: docs/요구사항.md

\- 작업 로그: docs/작업로그.md

\- 테스트 결과: docs/테스트결과.md

\- 서버 배포: docs/서버배포.md

\- 결정 사항: docs/결정사항.md
